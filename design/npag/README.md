# NPAG / NPB developer notes and golden-fixture provenance

The nonparametric engines validate against reference values captured from the
Pmetrics Rust implementation (crates `pmcore` 0.25.2 / `pharmsol` 0.26.1,
upstream `github.com/LAPKB/PMcore`).  CRAN cannot build or run Rust, so those
reference values are captured ONCE, offline, and committed as static fixtures
under `tests/testthat/fixtures/npag/`.  The test suite compares the C++ output
against those committed files only -- it never invokes Rust.

## Burke interior-point solver (`tests/testthat/fixtures/npag/ipm/burke-golden.csv`)

Generated by a standalone harness that reuses the VERBATIM `burke` source from
`pmcore` (`src/routines/estimation/ipm.rs`) over `faer` 0.24.0, so the numerics
are identical to Pmetrics.  Sources kept here for reproducibility:

- `burke-golden-gen.rs` -- the harness `main.rs`.
- `burke-golden-Cargo.toml` -- its `Cargo.toml`.

To regenerate:

```sh
mkdir -p burke_golden/src
cp design/npag/burke-golden-gen.rs   burke_golden/src/main.rs
cp design/npag/burke-golden-Cargo.toml burke_golden/Cargo.toml
( cd burke_golden && cargo run --release ) > \
  tests/testthat/fixtures/npag/ipm/burke-golden.csv
```

CSV schema (one row per record): `name,kind,idx,values...` where `kind` is
`psi` (idx = 0-based subject row), `weights` (idx = -1, one value per support
point), or `objective` (idx = -1, scalar log-likelihood).
