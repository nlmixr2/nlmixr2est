# RPEM -- Architecture

## Dispatch and R surface

- `nlmixr2Est.rpem` (`R/rpem.R`): S3 handler registered in the
  `R/nlmixr2Est.R` dispatch table. Mirrors `saem.R`'s structure.
- `rpemControl()` (`R/rpemControl.R`): control constructor; inherits
  `sharedControl()` common options, adds RPEM-specific knobs
  (`10-stopping-control.md`).
- UI -> rxode2 translation: `R/rpemRxUiGet.R` (+ model variant
  `R/rpemRxUiGetModel.R`) built via `rxPipeline()`. Reuse SAEM's handlers as the
  template; RPEM needs the same mu-referenced decomposition plus a classification
  of parameters into {random (eta), mixture, fixed}.
- Pre-processing: reuse `R/hook.R` chain -- bounded transforms
  (`preProcessBoundedTransform`), covariates, data/UI validation, zero-omega,
  literal fix. No RPEM-specific hook expected for M1.

## C++ kernel

- `src/rpem.cpp`: the E-step/M-step engine. Added to `SOURCES_CPP` in
  `src/Makevars(.in)`. Uses RcppArmadillo, OpenMP (`src/rxomp.h`), and rxode2's
  solve/likelihood C API.
- Exposed to R through `RcppExports` / `RcppExportMod` like `saem.cpp`.

## Data flow (one fit)

```
rpem UI --rxPipeline--> rxode2 model (compiled C)
        --hooks--> transformed/validated problem
   R: rpem.R assembles control + param classification + data matrices
   C++: rpem.cpp iterate:
        E-step: draw theta ~ mixture Gaussians
                --> rxode2 par_solve (batched population solve)
                --> rxode2 llik: p(Y_i | theta_i) per sample   [store]
                --> n_ik, N_i, tau, lnL
        M-step: joint MH over (i,k,theta) reusing stored likelihoods
                --> conjugate updates: mu^(k), Sigma^(k), w^(k)
                --> numeric updates (nlm): residual error + non-mu-ref fixed
        convergence check
   C++ --> estimates, lnL trace, converged-sample store (for SEs + predictions)
   R: build nlmixr2FitData; residuals/predictions lazily
```

## Reuse boundaries (interfaces, not copies)

- **Likelihood**: a **dedicated RPEM likelihood model** generated by a new
  `rxUiGet.rpemModel` handler -- a hybrid of `nlmModel0` (direct, no-optimization
  evaluation) and `foceiModel0ll` (THETA+ETA parameterization). RPEM supplies
  `theta_i` (population + eta on the transformed scale) as inputs and reads back
  the observation-level log-likelihood sum per subject/sample, with **no eta
  optimization**. Do not reimplement error-model math -- reuse
  `rxGetDistributionFoceiLines`/`rxCombineErrorLines`/`rxPredLlik`. Full design in
  `13-likelihood-model.md`.
- **ODE batch**: hand the full sample matrix to rxode2 as a population solve so
  `par_solve` threads over (subject x sample). Avoid per-sample R round-trips.
- **Numeric optimizer**: the M-step fixed-effect maximization reuses the
  nlm-family optimizer already wrapped for FOCEI/nlm.
- **Fit assembly**: reuse `nmObjGet`/`nmObjHandle` accessors and the SAEM fit
  assembly path (`saem_fit.R`, `saem_fit_aux.R`) as the template for
  `rpem_fit.R`.

## Non-goals for the engine

- No MPI (D3). No bespoke ODE solver (use rxode2's). No new residual-error math
  (use rxode2 llik). No new mixture UI grammar in M1 (K=1).
