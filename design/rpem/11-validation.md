# RPEM -- Validation and Test Plan

Four correctness bars (D12), all required. Correctness before speed (D1).

## Bar 1 -- Reproduce the paper's analytic case

- Model: paper's 1-compartment analytic PK, K=2 mixture (Eqs 34-39), paper's
  simulated data.
- Target: recover Table 1 true values within the paper's reported tolerances
  (e.g. `mu_V=20`, `w^(1)=0.8`, `sigma=0.1`, ...).
- Note: K=2 -> this bar depends on the mixture backend (`07-mixtures.md`), so it
  is an M2 acceptance item. An M1 smoke test uses a K=1 reduction of the same
  structural model.

## Bar 2 -- Match SAEM/FOCEI on nlmixr2 models

- Models: Warfarin PK (add+prop), Theophylline 1-cmt (D13).
- Target: RPEM population estimates, omega, residual params, and objective agree
  with existing SAEM and FOCEI fits within tolerance. Reuse cached fixtures
  (`tests/testthat/helper-zzz-fits.R`, `tests/testthat/fixtures/`) for the
  SAEM/FOCEI reference values.
- This is the primary M1 correctness gate (K=1, real error models, multiple
  endpoints where available).

## Bar 3 -- Recover simulation truth

- Simulate from a known model (via rxode2), fit with RPEM, confirm parameter
  recovery with acceptable bias and CI coverage (SEs from `08`).
- Run across a few seeds to check the "randomized" variance is controlled and
  the stopping rule is stable.

## Bar 4 -- Benchmark speed vs SAEM/FOCEI

- Model: voriconazole 3-ODE (paper, K=1) ported to nlmixr2, plus warfarin.
- Target: reproduce the paper's qualitative result (RPEM ~3-4x faster than SAEM,
  comparable accuracy) on comparable hardware/thread counts. Report wall-clock
  and accuracy side by side. Secondary to correctness.

## Test artifacts

- `tests/testthat/test-rpem-analytic.R` -- Bars 1 (K=1 smoke) / 3.
- `tests/testthat/test-rpem-warfarin.R`, `test-rpem-theo.R` -- Bar 2.
- `tests/testthat/test-rpem-vori.R` -- Bar 4 (guarded / slow, skip on CRAN).
- Add RPEM fits to the fixture pre-fit cache once stable.
- Respect the thread policy in `tests/testthat.R` (single worker on CI/CRAN;
  cap within-solve threads on CRAN).

## Invariants to assert regardless of model

- EM monotonicity: `lnL` non-decreasing across iterations (up to MC noise).
- Reproducibility for a fixed seed and fixed thread count.
- `sum_k w^(k) = 1`; `Sigma^(k)` positive-definite (via `nearPD` if repaired).
